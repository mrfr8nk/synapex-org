# Synapex Developers — Project Overview

## About
A premium marketing/portfolio website for "Synapex Developers" — a software agency. Built with React 19 + Vite (SPA), TanStack Router (client-side file-based routing), Tailwind CSS v4, and Framer Motion.

## Tech Stack
- **Framework:** React 19 + Vite 7 (pure SPA — no SSR)
- **Routing:** TanStack Router (client-side, file-based, auto-generated routeTree)
- **UI:** Tailwind CSS v4 + Radix UI primitives + Framer Motion animations
- **Package Manager / Runtime:** Bun
- **Build Tool:** Vite 7 with `@vitejs/plugin-react` + `@tanstack/router-plugin`
- **Optional CMS:** Supabase (graceful fallback to hardcoded content when env vars missing)

## Project Structure
```
src/
  main.tsx          # React app entry point (ReactDOM.createRoot)
  router.tsx        # TanStack Router config (createRouter)
  routeTree.gen.ts  # Auto-generated by TanStack Router plugin
  routes/           # File-based page routes (TanStack Router)
    __root.tsx      # App root route (Outlet only, no SSR shell)
    index.tsx       # Home page
    about.tsx, services.tsx, projects.tsx, pricing.tsx
    team.tsx, blog.tsx, blog.$slug.tsx, faq.tsx
    careers.tsx, contact.tsx, admin.tsx
  components/
    sections/       # Page sections (Hero, Services, Projects, Blog, etc.)
    ui/             # Reusable UI primitives (shadcn-style)
    Navbar.tsx, Footer.tsx, SiteLayout.tsx
  lib/
    content.ts      # Hardcoded fallback content
    useContent.ts   # Data hooks with Supabase + fallback
    admin.ts        # Admin auth helpers
  integrations/supabase/  # Supabase client + types
index.html          # SPA entry HTML
public/             # Static assets (logo, favicon)
supabase/           # Supabase config + migrations
```

## Admin Dashboard (`/admin`)
- Sidebar layout with navigation, Supabase status, sign out
- **Overview** — stats cards with counts per table
- **Content sections** — Services, Projects, Tech Stack, Clients, Testimonials, Team, Pricing, Blog Posts
  - Per-row **Visible/Hidden toggle** — hide any item from the frontend without deleting it
  - Image fields show Upload button + URL input (uses Supabase Storage `images` bucket)
- **Messages inbox** — contact form submissions with mark-read, reply via email, delete
- **Site Settings** — social links, contact info, footer tagline (stored in `site_content` table)

## Key Files
- `src/main.tsx` — React app entry (ReactDOM.createRoot + RouterProvider)
- `src/router.tsx` — TanStack Router setup (CSR-only)
- `src/lib/content.ts` — fallback data (used when Supabase not connected)
- `src/lib/useContent.ts` — data hooks; filters `visible !== false`, includes `saveSiteContentKey()`
- `src/components/Footer.tsx` — social links pulled from `useSiteContent()` dynamically
- `src/routes/admin.tsx` — full admin dashboard (sidebar layout, no SiteLayout wrapper)
- `supabase/migrations/` — SQL migrations including `visible` column + `site_content` table

## Dev Server
- Command: `bun run dev` → runs `vite`
- Vite default port (5173), `server.host: true`, `server.allowedHosts: true` for Replit proxy

## Scripts
- `bun run dev` — Start development server (Vite)
- `bun run build` — Production build (outputs to `dist/`)
- `bun run preview` — Preview production build locally

## Optional Supabase Setup
Set these environment variables for the CMS/admin features:
- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_PUBLISHABLE_KEY`

Without them, the site renders using hardcoded fallback content from `src/lib/content.ts`.

## Deployment
- Type: **Static Site** (pure SPA — no server needed)
- Build: `bun run build`
- Publish dir: `dist/`
- **SPA routing rule required on all hosts:** rewrite `/*` → `/index.html` (HTTP 200)
  - Render: add a Redirect/Rewrite rule in the Static Site settings
  - Vercel: add `vercel.json` with rewrites
  - Cloudflare Pages: add `public/_redirects`
